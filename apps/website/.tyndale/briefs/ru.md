# Translation Brief (en → ru)

## 1) App Context
- **Тип приложения:** маркетинговый сайт/лендинг и документационный интерфейс для **developer tool** (инструмента локализации).
- **Аудитория:** разработчики (React/Next.js), DevOps/CI пользователи, техлиды.
- **Суть продукта:** open-source инструмент (Tyndale) для автоматизации i18n-процесса: извлечение строк, перевод через AI-провайдера, инкрементальные обновления, интеграция в CLI/CI.
- **Контентный стиль:** короткие value-proposition блоки + CTA-кнопки + пошаговый onboarding.

---

## 2) Register & Formality
- Использовать **вежливую форму «вы»** (в UI — строчная: «вы»).
- Обращение должно быть **профессиональным и дружелюбным**, без панибратства.
- Для кнопок и призывов: **повелительное наклонение (мн. число)** или нейтральный инфинитив в шагах.
  - Пример CTA: **«Начните»**, **«Посмотреть на GitHub»**
  - Пример шага: **«Установить»**, **«Инициализировать»**

---

## 3) Tone
- **Тон:** профессиональный, технический, уверенный, лаконичный.
- **Не использовать:** разговорный сленг, чрезмерно «рекламные» формулировки, шутливость.
- **Должно звучать как:** современный SaaS/dev-tool с четкой выгодой и практическими действиями.
- **Ритм текста:** короткие фразы, высокая плотность смысла, минимум воды.

---

## 4) Term Decisions

### Оставлять в English (без перевода)
- **Бренды/имена:** `Tyndale`, `GitHub`, `React`, `Next.js`
- **Команды/кодовые токены:** ``tyndale validate``, `CLI`, `CI`, `JSX`, `RTL`, `i18n`
- **Технические аббревиатуры:** сохранять оригинал; при первом упоминании можно добавить русское пояснение при необходимости.

### Переводить на русский
- Общие продуктовые и маркетинговые элементы:
  - “Get Started” → **«Начните»**
  - “View on GitHub” → **«Посмотреть на GitHub»**
  - “Everything you need for i18n” → **«Всё, что нужно для i18n»**
  - “Open Source” → **«Открытый исходный код»** / **«Open source»** (предпочтительно единообразно выбрать один вариант; рекомендовано: **«Открытый исходный код»** в длинном тексте, **«Open source»** в коротких бейджах)
  - “AI-powered” → **«На базе ИИ»**
  - “Incremental” → **«Инкрементально»** или **«Инкрементальный режим»** (в зависимости от UI-контекста)
  - “Rich content” → **«Расширенный контент»** / **«Богатый контент»** (предпочтительно: **«Расширенный контент»**)
  - “Ship translated / Ship it” → **«Выпускайте с переводом»** / **«В продакшен»** (избегать буквального «шипить»)

### Термины, требующие консистентности
- **locale/locales** → **локаль/локали**
- **provider** → **провайдер**
- **static generation** → **статическая генерация**
- **middleware** → **middleware** (или «промежуточное ПО», но для dev-аудитории лучше оставить middleware)
- **workflow** → **процесс** / **workflow** (выбрать единообразно; рекомендовано: «процесс» в тексте, workflow — в тех. контексте)

## 5) Patterns

### CTA (кнопки, ссылки)
- **Основной паттерн:** короткий actionable текст.
- **Primary CTA:** повелительное наклонение  
  - “Get Started” → **«Начните»**
  - “Star on GitHub” → **«Поставьте звезду на GitHub»**
- **Шаги/этапы:** инфинитив (как чеклист/инструкция)  
  - “Install / Initialize / Translate” → **«Установить / Инициализировать / Перевести»**

### Error messages
- Тон: нейтральный, без обвинения пользователя.
- Структура: **что случилось + что делать дальше**.
  - Шаблон: **«Не удалось [действие]. Повторите попытку.»**
- Без восклицаний, без разговорных форм.

### Empty states
- Кратко объяснять отсутствие данных + следующий шаг.
  - Шаблон: **«Пока нет [объекта]. [Действие]»**
- Тон спокойный, ориентированный на действие.

### Confirmations/success
- Короткое подтверждение результата, по возможности с объектом.
  - Шаблон: **«Готово: [объект] обновлён.»**
- Можно добавлять следующий шаг, если релевантно.

### Questions
- Для маркетинговых вопросов использовать **«вы»** и мотивирующий, но деловой стиль.
  - “Ready to go global?” → **«Готовы выйти на глобальный рынок?»**
- Избегать излишне эмоциональных или кликбейтных формулировок.

--- 

**Ключевое правило:** держать интерфейс максимально понятным русскоязычному разработчику, сохраняя международные техтермины и точность исходного смысла.
