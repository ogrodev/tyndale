# Translation Brief (en → ko)

## 1) App Context
- **제품 유형:** 개발자 대상 **i18n(국제화) 자동화 도구**의 마케팅/랜딩 페이지 UI 카피
- **도메인:** React/Next.js 기반 앱 번역 워크플로(추출→번역→검증→배포), CLI/CI 연동, 오픈소스
- **주요 사용자:** 프론트엔드 개발자, DX/플랫폼 팀, 스타트업 엔지니어링 팀
- **카피 성격:** 기능 소개 + 빠른 시작 유도 + GitHub 전환(Star/Repo 방문)

---

## 2) Register & Formality
- **권장 레지스터:** **존댓말(합쇼체, -합니다/-하세요)**  
- **호칭/대명사:** `당신` 사용 금지. 2인칭은 생략하고 대상은 “앱/프로젝트/팀”으로 처리.
- **UI 라벨/버튼:** 문장형 존댓말 대신 **간결한 동작형(…하기/…보기)** 우선  
  - 예: “Get Started” → “시작하기”, “View on GitHub” → “GitHub에서 보기”

---

## 3) Tone
- **톤:** 전문적 + 기술 친화적 + 간결한 액션 중심
- **스타일:** 과장보다 신뢰성, 짧은 문장, 개발자 문맥 용어 자연스럽게 유지
- **금지 톤:** 과도하게 캐주얼한 유행어, 번역투 직역(부자연스러운 장문)
- **리듬:** 원문의 스피드감 유지 (짧은 헤드라인/스텝형 카피)

---

## 4) Term Decisions

### A. 영어 유지(브랜드/고유 기술명)
다음은 **원문 그대로 유지** 권장:
- **Tyndale**, **GitHub**, **React**, **Next.js**, **JSX**, **CLI**, **CI**, **RTL**

### B. 한국어 번역(또는 한국어 관용 표기)
- Docs → **문서** (네비게이션 정책상 일관되게 `Docs` 유지도 가능하나, ko 로캘에서는 “문서” 권장)
- Open Source / Open-source → **오픈 소스**
- AI-powered → **AI 기반**
- Zero-key workflow → **키 파일 없는 워크플로**
- Incremental → **증분(방식)** 또는 문맥상 **변경분만 번역**
- Rich content → **풍부한 콘텐츠**
- Translate → **번역하기**
- Ship it / Ship translated → **배포하기 / 번역된 상태로 배포하기**
- locale(s) → **로케일**
- static generation → **정적 생성**

### C. 핵심 용어 일관성
- i18n은 첫 노출 시 **“국제화(i18n)”**, 이후 **i18n** 단독 사용 가능
- provider는 문맥 따라 **프로바이더**(코드/아키텍처 맥락) 사용
- key files는 **키 파일**로 통일 (`키값 파일` 등 혼용 금지)

## 5) Patterns

### CTA (버튼/짧은 액션)
- **원칙:** 명령문보다 **행동 라벨형(…하기/…보기/…실행하기)**  
- 예:
  - Get Started → **시작하기**
  - Run the CLI → **CLI 실행하기**
  - See the result → **결과 보기**
  - Star on GitHub → **GitHub에서 스타 주기** (또는 제품 정책상 “Star” 유지: **GitHub에서 Star 주기**)

### 기능 설명/헤드라인
- 짧고 단정적으로: “~합니다”, “~할 수 있습니다”
- 예:  
  - “Only translates what changed…” → **마지막 실행 이후 변경된 내용만 번역합니다.**

### Error messages (향후 추가 시)
- **구조:** 문제 → 원인(선택) → 다음 행동
- **톤:** 비난 없이 정중, 해결 지시 포함
- 예: **번역을 완료하지 못했습니다. 설정을 확인한 뒤 다시 시도해 주세요.**

### Empty states (향후 추가 시)
- **구조:** 현재 상태 + 첫 액션 유도
- 예: **아직 번역 항목이 없습니다. CLI를 실행해 첫 번역을 생성하세요.**

### Confirmations (향후 추가 시)
- **완료형:** `~되었습니다` / `~완료`
- 예: **번역 검증이 완료되었습니다.**

### Questions
- 마케팅 질문: **-되셨나요? / -해볼까요?** (부드럽고 유도형)
  - Ready to go global? → **글로벌 출시할 준비가 되셨나요?**
- 설정/위험 동작 확인: **-하시겠습니까?** (명확·격식)
