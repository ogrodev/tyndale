## Convalida le traduzioni

`tyndale validate` controlla che tutte le lingue abbiano traduzioni complete e ben formate senza effettuare chiamate AI:

```bash
npx tyndale validate
```

Restituisce il codice 1 se vengono trovati problemi, rendendolo perfetto per la CI.

## GitHub Actions

```yaml
# .github/workflows/i18n.yml
name: i18n Validation
on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install
      - run: npx tyndale validate
```

## Hook pre-commit

Usalo con `husky` o `lint-staged`:

```json
{
  "lint-staged": {
    "*.{ts,tsx}": "tyndale validate"
  }
}
```

## Flusso di traduzione

1. Lo sviluppatore racchiude i contenuti con `<T>` o `msg()`
2. Esegui `npx tyndale translate` in locale (richiede credenziali AI)
3. Fai commit dei file di lingua generati in `public/_tyndale/`
4. La CI esegue `tyndale validate` per intercettare traduzioni mancanti
5. Rilascia con fiducia
