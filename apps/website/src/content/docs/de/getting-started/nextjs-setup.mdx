## Adapter installieren

```bash
npm install tyndale-next
```

## Konfiguration

<Steps>

1. **Aktualisiere deine Next.js-Konfiguration**

   ```js
   // next.config.mjs
   import { withTyndaleConfig } from 'tyndale-next/config';

   export default withTyndaleConfig({
     // your existing Next.js config
   });
   ```

2. **Füge die Middleware hinzu**

   ```ts
   // middleware.ts
   export { tyndaleMiddleware as middleware } from 'tyndale-next/middleware';
   ```

   Dies übernimmt die Locale-Erkennung, das URL-Routing und das Setzen des Locale-Cookies.

3. **Füge den Server-Provider zu deinem Layout hinzu**

   ```tsx
   // app/[locale]/layout.tsx
   import { TyndaleServerProvider } from 'tyndale-next/server';

   export default async function RootLayout({
     children,
     params,
   }: {
     children: React.ReactNode;
     params: Promise<{ locale: string }>;
   }) {
     const { locale } = await params;

     return (
       <html lang={locale}>
         <body>
           <TyndaleServerProvider locale={locale}>
             {children}
           </TyndaleServerProvider>
         </body>
       </html>
     );
   }
   ```

</Steps>

## So funktioniert es

- `withTyndaleConfig` fügt deiner Next.js-Konfiguration Locale-Pfad-Rewrites hinzu
- Die Middleware erkennt die bevorzugte Locale des Nutzers und leitet entsprechend um
- `TyndaleServerProvider` lädt Übersetzungen serverseitig und stellt sie allen Komponenten über den React-Kontext zur Verfügung
