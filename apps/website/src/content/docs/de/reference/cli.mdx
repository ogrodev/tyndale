## `tyndale init`

Initialisiert ein neues Tyndale-Projekt. Erstellt `tyndale.config.json` mit sinnvollen Standardwerten.

```bash
npx tyndale init
```

## `tyndale auth`

Konfiguriert Ihre Zugangsdaten für den AI-Provider.

```bash
npx tyndale auth
```

## `tyndale translate`

Extrahiert übersetzbare Strings und erzeugt Übersetzungen für alle konfigurierten Locales.

```bash
npx tyndale translate [options]
```

| Flag | Beschreibung |
|------|-------------|
| `--locale <code>` | Übersetzt nur ein bestimmtes Locale |
| `--force` | Übersetzt alle Strings erneut, nicht nur Änderungen |
| `--dry-run` | Zeigt, was übersetzt würde, ohne Dateien zu schreiben |
| `--token-budget <n>` | Token-Budget pro AI-Batch (Standard: 50000) |
| `--concurrency <n>` | Maximale parallele Übersetzungssitzungen (automatisch erkannt) |

## `tyndale validate`

Prüft, dass alle Locales vollständige, wohlgeformte Übersetzungen haben. Es werden keine AI-Aufrufe ausgeführt.

```bash
npx tyndale validate
```

Beendet sich mit Code 1, wenn Probleme gefunden werden. Ideal für CI-Pipelines.

## `tyndale extract`

Extrahiert übersetzbare Strings, ohne sie zu übersetzen. Nützlich, um zu prüfen, was an den AI-Provider gesendet wird.

```bash
npx tyndale extract
```


## `tyndale translate-docs`

Übersetzt Dokumentationsdateien (MDX/MD) für jedes unterstützte Framework. Erkennt Ihr Doku-Framework automatisch oder verwendet die Konfiguration aus `tyndale.config.json`.

```bash
npx tyndale translate-docs [options]
```

| Flag | Beschreibung |
|------|-------------|
| `--content-dir <path>` | Überschreibt das Content-Verzeichnis der Dokumentation |
| `--force` | Übersetzt alle Doku-Dateien erneut, nicht nur fehlende Dateien |
| `--concurrency <n>` | Maximale parallele Übersetzungssitzungen (automatisch erkannt) |

Unterstützte Frameworks: Starlight, Docusaurus, VitePress, MkDocs, Nextra.

### `tyndale translate-docs setup`

Erkennt Ihr Dokumentations-Framework und speichert die Konfiguration in `tyndale.config.json`.

```bash
npx tyndale translate-docs setup
```

Durchsucht Ihr Projekt nach Framework-Signalen (Paketabhängigkeiten und Konfigurationsdateien), bestätigt das erkannte Framework und schreibt den Abschnitt `docs` in Ihre Konfiguration.

## `tyndale model`

Ändert das für Übersetzungen verwendete AI-Modell.

```bash
npx tyndale model
```
