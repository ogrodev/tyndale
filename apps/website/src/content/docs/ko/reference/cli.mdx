## `tyndale init`

새 Tyndale 프로젝트를 초기화합니다. 합리적인 기본값으로 `tyndale.config.json`을 생성합니다.

```bash
npx tyndale init
```

## `tyndale auth`

AI provider 자격 증명을 구성합니다.

```bash
npx tyndale auth
```

## `tyndale translate`

번역 가능한 문자열을 추출하고 구성된 모든 로케일에 대한 번역을 생성합니다.

```bash
npx tyndale translate [options]
```

| 옵션 | 설명 |
|------|-------------|
| `--locale <code>` | 특정 로케일만 번역합니다 |
| `--force` | 변경된 항목만이 아니라 모든 문자열을 다시 번역합니다 |
| `--dry-run` | 파일을 쓰지 않고 번역될 내용을 보여줍니다 |
| `--token-budget <n>` | AI 배치당 토큰 예산 (기본값: 50000) |
| `--concurrency <n>` | 최대 병렬 번역 세션 수 (자동 감지) |

## `tyndale validate`

모든 로케일에 완전하고 올바른 형식의 번역이 있는지 확인합니다. AI 호출은 수행되지 않습니다.

```bash
npx tyndale validate
```

문제가 발견되면 종료 코드 1로 종료합니다. CI 파이프라인에 이상적입니다.

## `tyndale extract`

번역은 수행하지 않고 번역 가능한 문자열만 추출합니다. AI provider로 전송될 내용을 검토할 때 유용합니다.

```bash
npx tyndale extract
```


## `tyndale translate-docs`

지원되는 모든 프레임워크에서 문서 파일(MDX/MD)을 번역합니다. 문서 프레임워크를 자동으로 감지하거나 `tyndale.config.json`의 구성을 사용합니다.

```bash
npx tyndale translate-docs [options]
```

| 옵션 | 설명 |
|------|-------------|
| `--content-dir <path>` | 문서 콘텐츠 디렉터리를 재정의합니다 |
| `--force` | 누락된 파일만이 아니라 모든 문서를 다시 번역합니다 |
| `--concurrency <n>` | 최대 병렬 번역 세션 수 (자동 감지) |

지원되는 프레임워크: Starlight, Docusaurus, VitePress, MkDocs, Nextra.

### `tyndale translate-docs setup`

문서화 프레임워크를 감지하고 구성을 `tyndale.config.json`에 저장합니다.

```bash
npx tyndale translate-docs setup
```

프로젝트에서 프레임워크 신호(패키지 의존성과 구성 파일)를 스캔하고, 감지된 프레임워크를 확인한 뒤, 구성 파일에 `docs` 섹션을 작성합니다.

## `tyndale model`

번역에 사용할 AI 모델을 변경합니다.

```bash
npx tyndale model
```
