## 翻訳を検証する

`tyndale validate` は、AI 呼び出しを一切行わずに、すべてのロケールに完全で正しい形式の翻訳があるかを確認します:

```bash
npx tyndale validate
```

問題が見つかった場合は終了コード 1 で終了するため、CI に最適です。

## GitHub Actions

```yaml
# .github/workflows/i18n.yml
name: i18n Validation
on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install
      - run: npx tyndale validate
```

## pre-commit フック

`husky` または `lint-staged` と併用します:

```json
{
  "lint-staged": {
    "*.{ts,tsx}": "tyndale validate"
  }
}
```

## 翻訳ワークフロー

1. 開発者がコンテンツを `<T>` または `msg()` でラップする
2. ローカルで `npx tyndale translate` を実行する（AI 認証情報が必要）
3. 生成されたロケールファイルを `public/_tyndale/` にコミットする
4. CI で `tyndale validate` を実行して翻訳漏れを検出する
5. 安心してリリースする
