## `tyndale init`

Инициализирует новый проект Tyndale. Создаёт `tyndale.config.json` с разумными значениями по умолчанию.

```bash
npx tyndale init
```

## `tyndale auth`

Настраивает учётные данные вашего AI-провайдера.

```bash
npx tyndale auth
```

## `tyndale translate`

Извлекает переводимые строки и генерирует переводы для всех настроенных локалей.

```bash
npx tyndale translate [options]
```

| Флаг | Описание |
|------|-------------|
| `--locale <code>` | Переводить только указанную локаль |
| `--force` | Повторно перевести все строки, а не только изменения |
| `--dry-run` | Показать, что будет переведено, без записи файлов |
| `--token-budget <n>` | Бюджет токенов на пакет AI (по умолчанию: 50000) |
| `--concurrency <n>` | Максимальное число параллельных сессий перевода (определяется автоматически) |

## `tyndale validate`

Проверяет, что у всех локалей есть полные и корректно сформированные переводы. Вызовы AI не выполняются.

```bash
npx tyndale validate
```

Завершается с кодом 1, если обнаружены проблемы. Идеально для CI-конвейеров.

## `tyndale extract`

Извлекает переводимые строки без их перевода. Полезно для проверки того, что будет отправлено AI-провайдеру.

```bash
npx tyndale extract
```
