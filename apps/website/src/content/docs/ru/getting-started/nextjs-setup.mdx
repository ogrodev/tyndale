## Установите адаптер

```bash
npm install tyndale-next
```

## Конфигурация

<Steps>

1. **Обновите конфигурацию Next.js**

   ```js
   // next.config.mjs
   import { withTyndaleConfig } from 'tyndale-next/config';

   export default withTyndaleConfig({
     // your existing Next.js config
   });
   ```

2. **Добавьте middleware**

   ```ts
   // middleware.ts
   export { tyndaleMiddleware as middleware } from 'tyndale-next/middleware';
   ```

   Это обрабатывает определение локали, маршрутизацию URL и установку cookie локали.

3. **Добавьте server provider в ваш layout**

   ```tsx
   // app/[locale]/layout.tsx
   import { TyndaleServerProvider } from 'tyndale-next/server';

   export default async function RootLayout({
     children,
     params,
   }: {
     children: React.ReactNode;
     params: Promise<{ locale: string }>;
   }) {
     const { locale } = await params;

     return (
       <html lang={locale}>
         <body>
           <TyndaleServerProvider locale={locale}>
             {children}
           </TyndaleServerProvider>
         </body>
       </html>
     );
   }
   ```

</Steps>

## Как это работает

- `withTyndaleConfig` добавляет перезапись путей локалей в конфигурацию Next.js
- Middleware определяет предпочитаемую пользователем локаль и выполняет соответствующий редирект
- `TyndaleServerProvider` загружает переводы на стороне сервера и делает их доступными всем компонентам через React context
