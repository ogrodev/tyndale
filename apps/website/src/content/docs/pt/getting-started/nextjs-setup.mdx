## Instale o adaptador

```bash
npm install tyndale-next
```

## Configuração

<Steps>

1. **Atualize sua configuração do Next.js**

   ```js
   // next.config.mjs
   import { withTyndaleConfig } from 'tyndale-next/config';

   export default withTyndaleConfig({
     // your existing Next.js config
   });
   ```

2. **Adicione o middleware**

   ```ts
   // middleware.ts
   export { tyndaleMiddleware as middleware } from 'tyndale-next/middleware';
   ```

   Isso cuida da detecção de localidade, do roteamento de URL e da definição do cookie de localidade.

3. **Adicione o provedor de servidor ao seu layout**

   ```tsx
   // app/[locale]/layout.tsx
   import { TyndaleServerProvider } from 'tyndale-next/server';

   export default async function RootLayout({
     children,
     params,
   }: {
     children: React.ReactNode;
     params: Promise<{ locale: string }>;
   }) {
     const { locale } = await params;

     return (
       <html lang={locale}>
         <body>
           <TyndaleServerProvider locale={locale}>
             {children}
           </TyndaleServerProvider>
         </body>
       </html>
     );
   }
   ```

</Steps>

## Como funciona

- `withTyndaleConfig` adiciona reescritas de caminho de localidade à sua configuração do Next.js
- O middleware detecta a localidade preferida do usuário e redireciona de acordo
- `TyndaleServerProvider` carrega as traduções no servidor e as disponibiliza para todos os componentes via contexto do React
