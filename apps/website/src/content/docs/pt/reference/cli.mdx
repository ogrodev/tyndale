## `tyndale init`

Inicializa um novo projeto Tyndale. Cria `tyndale.config.json` com padrões sensatos.

```bash
npx tyndale init
```

## `tyndale auth`

Configure as credenciais do seu provedor de IA.

```bash
npx tyndale auth
```

## `tyndale translate`

Extrai strings traduzíveis e gera traduções para todos os locales configurados.

```bash
npx tyndale translate [options]
```

| Opção | Descrição |
|------|-------------|
| `--locale <code>` | Traduz apenas um locale específico |
| `--force` | Retraduz todas as strings, não apenas as alterações |
| `--dry-run` | Mostra o que seria traduzido sem gravar arquivos |
| `--token-budget <n>` | Orçamento de tokens por lote de IA (padrão: 50000) |
| `--concurrency <n>` | Máximo de sessões de tradução paralelas (detectado automaticamente) |

## `tyndale validate`

Verifica se todos os locales têm traduções completas e bem formadas. Nenhuma chamada de IA é feita.

```bash
npx tyndale validate
```

Encerra com código 1 se forem encontrados problemas. Ideal para pipelines de CI.

## `tyndale extract`

Extrai strings traduzíveis sem traduzi-las. Útil para revisar o que será enviado ao provedor de IA.

```bash
npx tyndale extract
```


## `tyndale translate-docs`

Traduz arquivos de documentação (MDX/MD) para qualquer framework suportado. Detecta automaticamente o framework de documentação ou usa a configuração de `tyndale.config.json`.

```bash
npx tyndale translate-docs [options]
```

| Opção | Descrição |
|------|-------------|
| `--content-dir <path>` | Sobrescreve o diretório de conteúdo da documentação |
| `--force` | Retraduz toda a documentação, não apenas os arquivos ausentes |
| `--concurrency <n>` | Máximo de sessões de tradução paralelas (detectado automaticamente) |

Frameworks suportados: Starlight, Docusaurus, VitePress, MkDocs, Nextra.

### `tyndale translate-docs setup`

Detecta seu framework de documentação e salva a configuração em `tyndale.config.json`.

```bash
npx tyndale translate-docs setup
```

Analisa seu projeto em busca de sinais do framework (dependências de pacote e arquivos de configuração), confirma o framework detectado e grava a seção `docs` na sua configuração.

## `tyndale model`

Altera o modelo de IA usado nas traduções.

```bash
npx tyndale model
```
