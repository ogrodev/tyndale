## Installer l’adaptateur

```bash
npm install tyndale-next
```

## Configuration

<Steps>

1. **Mettez à jour votre configuration Next.js**

   ```js
   // next.config.mjs
   import { withTyndaleConfig } from 'tyndale-next/config';

   export default withTyndaleConfig({
     // your existing Next.js config
   });
   ```

2. **Ajoutez le middleware**

   ```ts
   // middleware.ts
   export { tyndaleMiddleware as middleware } from 'tyndale-next/middleware';
   ```

   Cela gère la détection de la locale, le routage des URL et la définition du cookie de locale.

3. **Ajoutez le provider serveur à votre layout**

   ```tsx
   // app/[locale]/layout.tsx
   import { TyndaleServerProvider } from 'tyndale-next/server';

   export default async function RootLayout({
     children,
     params,
   }: {
     children: React.ReactNode;
     params: Promise<{ locale: string }>;
   }) {
     const { locale } = await params;

     return (
       <html lang={locale}>
         <body>
           <TyndaleServerProvider locale={locale}>
             {children}
           </TyndaleServerProvider>
         </body>
       </html>
     );
   }
   ```

</Steps>

## Fonctionnement

- `withTyndaleConfig` ajoute des réécritures de chemin de locale à votre configuration Next.js
- Le middleware détecte la locale préférée de l’utilisateur et redirige en conséquence
- `TyndaleServerProvider` charge les traductions côté serveur et les rend disponibles pour tous les composants via le contexte React
