## `tyndale init`

Initialisez un nouveau projet Tyndale. Crée `tyndale.config.json` avec des valeurs par défaut pertinentes.

```bash
npx tyndale init
```

## `tyndale auth`

Configurez les identifiants de votre fournisseur d’IA.

```bash
npx tyndale auth
```

## `tyndale translate`

Extrait les chaînes traduisibles et génère des traductions pour toutes les locales configurées.

```bash
npx tyndale translate [options]
```

| Option | Description |
|------|-------------|
| `--locale <code>` | Traduit uniquement une locale spécifique |
| `--force` | Retraduit toutes les chaînes, pas seulement les modifications |
| `--dry-run` | Affiche ce qui serait traduit sans écrire de fichiers |
| `--token-budget <n>` | Budget de tokens par lot IA (par défaut : 50000) |
| `--concurrency <n>` | Nombre maximal de sessions de traduction parallèles (détecté automatiquement) |

## `tyndale validate`

Vérifie que toutes les locales disposent de traductions complètes et bien formées. Aucun appel à l’IA n’est effectué.

```bash
npx tyndale validate
```

Se termine avec le code 1 si des problèmes sont détectés. Idéal pour les pipelines CI.

## `tyndale extract`

Extrait les chaînes traduisibles sans les traduire. Utile pour examiner ce qui sera envoyé au fournisseur d’IA.

```bash
npx tyndale extract
```
