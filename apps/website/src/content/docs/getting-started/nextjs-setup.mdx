## Install the adapter

```bash
npm install tyndale-next
```

## Configuration

<Steps>

1. **Update your Next.js config**

   ```js
   // next.config.mjs
   import { withTyndaleConfig } from 'tyndale-next/config';

   export default withTyndaleConfig({
     // your existing Next.js config
   });
   ```

2. **Add the middleware**

   ```ts
   // middleware.ts
   export { tyndaleMiddleware as middleware } from 'tyndale-next/middleware';
   ```

   This handles locale detection, URL routing, and setting the locale cookie.

3. **Add the server provider to your layout**

   ```tsx
   // app/[locale]/layout.tsx
   import { TyndaleServerProvider } from 'tyndale-next/server';

   export default async function RootLayout({
     children,
     params,
   }: {
     children: React.ReactNode;
     params: Promise<{ locale: string }>;
   }) {
     const { locale } = await params;

     return (
       <html lang={locale}>
         <body>
           <TyndaleServerProvider locale={locale}>
             {children}
           </TyndaleServerProvider>
         </body>
       </html>
     );
   }
   ```

</Steps>

## How it works

- `withTyndaleConfig` adds locale path rewrites to your Next.js config
- The middleware detects the user's preferred locale and redirects accordingly
- `TyndaleServerProvider` loads translations server-side and makes them available to all components via React context
