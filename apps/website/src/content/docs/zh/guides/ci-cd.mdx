## 验证翻译

`tyndale validate` 会检查所有 locale 是否具备完整且格式正确的翻译，并且不会发起任何 AI 调用：

```bash
npx tyndale validate
```

如果发现任何问题，它会以退出码 1 结束，因此非常适合用于 CI。

## GitHub Actions

```yaml
# .github/workflows/i18n.yml
name: i18n Validation
on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install
      - run: npx tyndale validate
```

## Pre-commit 钩子

可配合 `husky` 或 `lint-staged` 使用：

```json
{
  "lint-staged": {
    "*.{ts,tsx}": "tyndale validate"
  }
}
```

## 翻译工作流

1. 开发者用 `<T>` 或 `msg()` 包裹内容
2. 在本地运行 `npx tyndale translate`（需要 AI 凭据）
3. 提交在 `public/_tyndale/` 中生成的 locale 文件
4. CI 运行 `tyndale validate` 以捕获缺失的翻译
5. 自信发布
